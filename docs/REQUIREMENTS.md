# FlowFit — Features & Requirements (Client Reference)

> This is a mirror of the API requirements for frontend context. See [flow-fit-api/docs/REQUIREMENTS.md](../../flow-fit-api/docs/REQUIREMENTS.md) for the full specification.

## Product Overview

**FlowFit** — CRM для фітнес-тренера. Мобільний додаток для управління розкладом, клієнтами та аналітикою.

- **Платформа:** Ionic 8 + Angular 20 (PWA, mobile-first)
- **Мова інтерфейсу:** Українська
- **Тема:** Світла (тепла) з палітрою: Peach #FFB3A7, Magenta #B55C82, Olive #829368

---

## Feature List

### F1: Аутентифікація
- Реєстрація тренера (email + пароль)
- Логін
- Автоматичне оновлення JWT токена (refresh)
- Перенаправлення на логін при expired token

### F2: Управління клієнтами
- Список клієнтів з пошуком та фільтрами
- Створення нового клієнта (модальне вікно)
- Редагування клієнта
- Архівація (soft-delete) клієнта
- Картка клієнта з вкладками:
  - **Інфо** — ПІБ, телефон, ціль, вага, коментарі
  - **Прогрес** — графік ваги, історія метрик
  - **Нотатки** — текст + посилання (Google Drive, Notion)
  - **Історія** — минулі та заплановані тренування

### F3: Управління локаціями
- Список студій/залів
- CRUD локацій (назва, тип, адреса)

### F4: Розклад тренувань (Scheduler)
- **Режим списку** (List View) — хронологічна стрічка тренувань
- **Режим календаря** (Calendar View) — тижневий/місячний
- Створення тренування:
  - Тип: індивідуальне / групове
  - Локація, дата/час, ціна
- Додавання учасників:
  - Існуючий клієнт з бази
  - Текстове ім'я (для разових гостей)
- Зміна статусу тренування (кнопки швидких дій)
- Фільтри: тип, статус, локація, клієнт
- Дублювання попереднього тижня

### F5: Звіти та аналітика
- Дохід за період (тиждень/місяць)
- Розбивка: індивідуальні vs групові
- Кількість проведених годин
- Відсоток пропусків
- Загальна статистика (кількість клієнтів, середня ціна)

### F6: Налаштування
- Профіль тренера (ім'я, email)
- Прив'язка Telegram (інструкція + статус)
- Налаштування нагадувань (за 3 год / за 1 год)
